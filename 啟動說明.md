## 檔案說明

| 檔案 | 說明 |
|------|------|
| `healing-emergency-room.html` | 網站前端（主頁面） |

## 第一次啟動（安裝依賴）

打開終端機（Terminal），進入網站資料夾，執行：

```bash
pip3 install flask flask-cors requests
```

---

## 每次啟動步驟

### 第一步：啟動後端伺服器

```bash
python3 server.py
```

看到以下訊息即代表成功：

```
==================================================
🌿 小Man 療癒急救室 後端伺服器
==================================================
✅ 使用模型：glm-4-plus
🌐 打開瀏覽器訪問：http://localhost:8888
==================================================
```

### 第二步：打開網站

方法 A（推薦）：直接在瀏覽器輸入：
```
http://localhost:8888
```

方法 B：雙擊打開 `healing-emergency-room.html`，但小Man 需要伺服器才能回應。

---

## 常見問題

**Q：小Man 沒有回應？**
- 確認 `server.py` 正在運行
- 瀏覽器頂部應該有橙色提示框說明伺服器未啟動
- 重新啟動 `server.py` 後，重新整理頁面

**Q：啟動時出現「Port already in use」？**
```bash
# 找出佔用 8888 端口的程式並終止
lsof -i :8888
kill -9 [PID]
```
然後重新執行 `python3 server.py`

**Q：pip3 找不到？**
```bash
pip install flask flask-cors requests
```

## 技術說明

- 後端使用 **Python Flask**，監聽 `localhost:8888`
- 前端通過 `http://localhost:8888/api/chat` 發送訊息
- 伺服器將請求轉發到 **Z.ai API**（GLM-4-Plus 模型）
- 小Man 的完整系統提示（訓練資料）存放在 `server.py` 的 `SYSTEM_PROMPT` 變數中
- API 金鑰已預設置於 `server.py`，無需另行設定
